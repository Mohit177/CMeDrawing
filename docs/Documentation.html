Computer Graphics- Assignment 1

L-Systems or Lindenmayer Systems are parallel rewriting systems and a type of formal grammar. The recursive nature of L-Systems makes it possible for beautiful fractals to be drawn out of simple rules applied recursively over multiple iterations. Plant models and organic forms can be easily defined and increasing recursion will grow them into complex structures. The property of “The closer you get, the more they look the same” is truly remarkable.

To implement the tasks, the following functions were used.
Task 1 (Line Drawing) : The drawLine() function of primitives.cpp was used for drawing a line between two points. The bresenham’s midpoint line drawing algorithm is used. Additionally required thickness can also be given as parameter.

Task 2 (Circle Drawing) : The drawCircle() function of primitives.cpp was used for drawing a circle given coordinates of center and radius. Midpoint circle drawing algorithm used. Requires color to be a parameter.

lineCircle.cpp is a menu driven program to freely draw lines and circles. (For first two tasks)

Task 3 (LSystems) : We tried two types of LSystems:- class LSystem1 : For simple branching and recursion. class LSystem2: For Turtle drawing.

Given the set of rules (std::map >), and axiom (char or std::string), generateString() produces a string that specifies drawing instructions to the turtle. generatePset() produces std::vector > pset, where coords is a data structure representing a pair of points with thickness and color (for drawing a line between them). This list in a way represents the line drawing commands which are iterated and called upon by drawLine() function. Some drawings might cause the figures points to go outside the window bounds. To overcome this problem, confineToViewPort() transforms all the points in LSystem2::pset to a given bound within the window bounds. Say a figure that lies between (500,500) and (2000,2000) can be transformed to the viewPort (100,100) to (200,200).

We found that Turtle drawing implementation could represent all those that the former could, but not the other way round.

Project Design

We created a private repository in GitHub accessible and visible only to the two of us, so that we could easily collaborate on the assignment.

The key components of the project folder include:

include folder
This folder contains all the required classes to create primitives and use them to make the complex figure.
The folder contains the following files:
1. context.h - Contains class for creating window and frame buffer.
2. context.cpp - Constructor and member functions of Context class are defined here.
3. primitives.h - Contains Primitive class for drawing primitives, which on using repetitively produces the required complex figure. (Task 1 and Task2)
4. primitives.cpp - Contains implementation of point drawing, line drawing, circle drawing, translate, rotate, viewPortTransform, Definitions for constructor and member functions in primitives.h
5. lsystem.h - Contains classes LSystem1 and LSystem2 for two types of LSystems.
6. lsystem.cpp - Contains definition of generatePset(), generateString, confineToViewPort() for constructing our drawing.

lineCircle.cpp - Menu driven program to use only line and circle drawing.
dInstr.txt - Specifies drawing instructions to main.cpp
Contains information of the format:
-------------------------------------------------------------------------------------------------------------
First Line contains no of Graftals to be drawn (t)

For each of the t-cases:
{no. of rules:int} {axiom:string}
Next N lines have rules described like:
{nonterminal:char}:{rule:string}
Then next line contains,
{no_of_iterations:int} {angle:int} {length:int} {thickness:int}
{starting point x:double} {starting point y:double}
{type}
if(type==1)
Follows input {Lower bound x} {Lower Bound y} {Upper bound x} {Upper Bound y} all double type.
else if(type==0)
Nothing
--------------------------------------------------------------------------------------------------------------
The file: dInstr.txt must be fed with multiple test cases. Each test requires rules, angle, length, no_of_iterations, thickness, starting point and whether viewPort transformation required. If so, the value of viewPort bounds.

main.cpp
Creates Context and Primitive class objects to set up window of 1200x700 pixels. Reads from
the file dInstr.txt which describes the drawing instructions. For each test case an object of LSystem2 is created along with drawing parameters. p.drawLine() (p object of class Primitive)
Is called for each pair of points in the list pset, which draws all the required lines.

Designs folder
Each file in this folder contains grammar rules for an L-System design.

testing folder
This folder contains the following file
draw.cpp - Generates dInstr.txt for the main.cpp in testing folder to draw
main.cpp - Executes for the test cases generated by draw.cpp and makes file sample.txt with execution times for each iteration. main.py - To generate plot.

docs folder
Contains documentation in html format. This site (https://word2cleanhtml.com/) was used to convert docx file into html and in addition to the convertor many lines were manually added.

Analysis Report

The generateString() function and generatePset() function were tested for their execution time against level of L-System ( number of iterations ). It took about 1 second for iteration 8 and took more than 10 seconds for iteration 9. A plot was constructed with a csv generated of the form Outcome of the plot generated using python library matplotlib.pyplot:

We can infer the time complexity grows exponentially with increase in the number of iterations.
For a string of length k generated by generateString() : Time complexity for generatePset() is O(k).

We can infer the time complexity grows exponentially with increase in the number of iterations. For a string of length k generated by generateString() : Time complexity for generatePset() is O(k).

For Line drawing algorithm: Loop runs from xi,yi to xf,yf: Time complexity = O(max(|xf-xi|,|yf-yi|)
For Circle drawing algorithm: Center x,y and radius r: Time complexity = O(8*radius)
